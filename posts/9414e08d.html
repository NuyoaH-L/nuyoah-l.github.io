Hexo主题优化之旅

本文总阅读量 加载中... 次 2026年05月25日 约545字 预计需要2分钟 blog

博客修改汇总 修改日期：2026-05-25

版权信息迁移

背景： 原版权信息位于侧栏底部，现移出侧栏，分别展示在首页和文章页的合理位置。

改动清单：

文件 操作 说明
layout/_partial/site-copyright.ejs 新建 独立版权 Partial，仅含版权文字
layout/_layout/nexmoe/header.ejs 修改 移除版权块，仅保留 <%- theme.slotSidebar %>
layout/index.jsx 修改 分页器后引入版权 Partial
layout/post.ejs 修改 评论组件后引入版权 Partial
source/css/style.styl 修改 添加全局 .nexmoe-copyright 样式
source/css/_layout/nexmoe/header.styl 修改 移除侧栏内的 .nexmoe-copyright 样式

效果：

  • 首页：文章列表 → 分页器 → 版权信息
  • 文章页：文章正文 → 标签 → 评论区 → 版权信息
  • 侧栏：仅保留 slotSidebar，不再显示版权文字

首页 SEO 优化

背景： 首页列表每篇文章标题均为 <h1>，违反每页仅一个 <h1> 的最佳实践。

改动清单：

文件 操作 说明
layout/_index/list.ejs 修改 文章标题 <h1> → <h2>（2 处）
source/css/_index/list.styl 修改 CSS 选择器更新，同时匹配 h1, h2
layout/index.jsx 修改 文章列表前添加 sr-only 的 <h1>，内容为站点描述
source/css/style.styl 修改 添加 .nexmoe-sr-only 屏幕阅读器样式

效果：

  • 首页 DOM：<h1 class="nexmoe-sr-only">（对用户隐藏）→ <h2>文章标题</h2> …
  • 文章页 <h1> 保持不变

右侧工具栏修复

背景： CSS 编辑时丢失了桌面端代码，导致文章页右下角的目录按钮和回到顶部按钮不可见。

改动清单：

文件 操作
source/css/style.styl 恢复 .nexmoe-post-right .nexmoe-fixed、.nexmoe-post-right .nexmoe-tool 等桌面端定位样式

样式微调

改动：

文件 说明
source/css/style.styl 版权字体 12px → 16px，增加上方分割线 border-top
source/css/_index/list.styl 封面标题 CSS 选择器从 h1 扩展为 h1, h2，兼容首页和文章页
